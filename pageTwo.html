Playing with Twitter's REST and STREAMING API's

Now that we have what we need to get your started, you may be wondering what exactly an API is. API simply stands for Application Programming Interface, which is essentially a complex name for a simple concept. Oversimplified, all an API is, is an interface that allows individual programs (or users) to interact with certain features of another program. It is essentially a form that dictates a form of communication between on program and another. This allow for new and interesting things to happen as one program can communicate and receive data from another.


If you want a more detailed explanation of how it all works here is a video on youtube that goes into a much more detailed explanation.




API’s are important because they contribute to the cool things that you can do or what you see on the web. We will be using the twit library to help us use Twitters REST and STREAMING API. To quickly differentiate the two, the REST and STREAMING API is very simple. The streaming API is essentially a continuous request to Twitter’s servers via an HTTP connection. All this means is that there is continuous communication between Twitter and your program, which allows for cool things to happen during individual “events” such as instant replies if someone follows or mentions you. The REST API essentially executes a command and terminates communication until the next request is sent to Twitter.This means you can send a request for data, such as searching for individual tweets or keywords and Twitter will return the information to you and wait for the next request.
